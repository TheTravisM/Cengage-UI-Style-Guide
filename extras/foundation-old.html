Document Overview About this document

This document describes web standards for Cengage Learning corporate web pages and sites. The guidelines also address usage of global elements — such as footers — for all Cengage Learning websites.

This document contains guidelines for overall page templates, specifications for individual design elements, and best practices. These guidelines ensure a consistent presentation of Cengage Learning and compliance with our accessibility standards.

The Quick Links appendix is a list of resources, links, templates and information to support specifications or requirements mentioned throughout this document.

Supported Browsers and Platforms

Browser usage on Cengage Learning properties are assessed each quarter to determine an appropriate support level (see Quick Links appendix).

Supported browsers and devices vary based on the specific website objectives.

Requirements

  • Web pages must display and function correctly in all browsers and platforms, including mobile devices, identified as “supported” per current guidelines (see Quick Links appendix)
  • Web pages must meet the Cengage Learning target compliance level of WCAG 2.0 Level AA for accessibility (see Accessibility appendix)
  • Elements using CSS or JavaScript must gracefully degrade for the older or less capable browsers in our supported list to still allow access to the basic content although the appearance may change. For example, CSS curved buttons may appear square cornered in older versions of IE
  • Use CSS3 or JavaScript for transitions and animations. When possible, CSS3 is preferred for better performance and simplicity
  • Flash is not supported and should not be used as a page element
  • To ensure accessibility, animations or multimedia must include viewer controls, a pause and/or stop button and must follow Cengage Learning Marketing Video Guidelines (see Quick Links appendix)

Font (HTML)

CSS Cascades as follows:

font-family: Arial, Helvetica, sans-serif

Requirements

  • Use the Cengage Learning Marketing CSS to ensure that fonts and sizes display consistent with the brand standards
  • Use auto-hyphenation only if needed
  • The term “Cengage Learning” should use a non-breaking space character to keep the two words together
  • Avoid blinking or moving (marquee) text
  • Maximum line length should be 100–120 characters for on-screen readability, ideally slightly shorter at 55-75 characters per line

Text Within Graphics

Limit text in banner graphics to approximately 200 characters for brand identification, a value statement and the call to action. This character count will fit comfortably in the area and allow sufficient reading time in a rotating banner.

Use Kievit, which is our corporate font, or Arial.

In Photoshop, use “crisp” anti-alias setting with a tracking value of 20.

Sample banner graphic with text

Requirements

  • For accessibility, use text rendered as graphics sparingly since it will appear blurry to lowvision users employing magnification tools
  • All graphic text must have the appropriate ALT and title attributes for accessibility
  • Graphics are not to be used for navigation
  • Ensure that text and graphics when viewed without color are accessible, meeting WCAG 2.0 Level AA standards (see Accessibility appendix)

Photography and Imagery

Images and photography help Cengage.com maintain a cohesive look.

Images should be unique and impactful:

  • Select simple, bold images with a single focus
  • Choose metaphorical rather than literal images whenever possible
  • Use easy-to-comprehend images that fit the message
  • Use clean backgrounds that don’t distract from the main image

Guidelines for stock photography:

  • Don’t use stock photos of people. Instead, obtain original images of customers, authors, etc. People photos should convey the scope and diversity of the Cengage Learning universe.
  • When using non-people stock photos, avoid clichés. Look for unique or unusual photos.

Home page promotions

The home page may feature one or more large background images. One “hero” with a single focus and message is displayed per visit or screen refresh. The option for a background image is reserved for the home page. HTML text is preferred for accessibility when using a single static image.

Specifications

  • Home page hero images fit 975 px x 470 px, with copy and critical image content in the top 280 px to clear the secondary message overlay band (126 px tall). Images may extend beneath the overlay bar to 2000 px x 1000 px for visual impact on larger screens
  • JavaScript code is used to display one image randomly from a set each time the page reloads or refreshes (see Quick Links appendix)

Hero sample:

Inner magenta border shows default visual area between header and footer (975 px x 470) on a typical screen size. Outer magenta line is full available area for expanded image background to fill larger screens. The tinted band area is the optional secondary message overlay area.

Secondary pages

A banner graphic conveys a single message; a slideshow script displays multiple messages or images. Page banners should support the page content and don't necessarily have a call to action.

Specifications

  • Secondary pages may have a static image or slideshow that fits the page grid. Full width is 975 px with a suggested depth of 224 px, or with left column navigation, 724 px x 224 px
  • Limit slideshows to a maximum of five slides
  • Headshots, like an author photo, should be 120 px x 150 px
