Overview About this document

This document describes web standards for Cengage Learning corporate web pages and sites. The guidelines also address usage of global elements — such as footers — for all Cengage Learning websites.

This document contains guidelines for overall page templates, specifications for individual design elements, and best practices. These guidelines ensure a consistent presentation of Cengage Learning and compliance with our accessibility standards.

The Quick Links appendix is a list of resources, links, templates and information to support specifications or requirements mentioned throughout this document.

Supported Browsers and Platforms

Browser usage on Cengage Learning properties are assessed each quarter to determine an appropriate support level (see Quick Links appendix).

Supported browsers and devices vary based on the specific website objectives.

Requirements

  • Web pages must display and function correctly in all browsers and platforms, including mobile devices, identified as “supported” per current guidelines (see Quick Links appendix)
  • Web pages must meet the Cengage Learning target compliance level of WCAG 2.0 Level AA for accessibility (see Accessibility appendix)
  • Elements using CSS or JavaScript must gracefully degrade for the older or less capable browsers in our supported list to still allow access to the basic content although the appearance may change. For example, CSS curved buttons may appear square cornered in older versions of IE
  • Use CSS3 or JavaScript for transitions and animations. When possible, CSS3 is preferred for better performance and simplicity
  • Flash is not supported and should not be used as a page element
  • To ensure accessibility, animations or multimedia must include viewer controls, a pause and/or stop button and must follow Cengage Learning Marketing Video Guidelines (see Quick Links appendix)

Font (HTML)

CSS Cascades as follows:

font-family: Arial, Helvetica, sans-serif

Requirements

  • Use the Cengage Learning Marketing CSS to ensure that fonts and sizes display consistent with the brand standards
  • Use auto-hyphenation only if needed
  • The term “Cengage Learning” should use a non-breaking space character to keep the two words together
  • Avoid blinking or moving (marquee) text
  • Maximum line length should be 100–120 characters for on-screen readability, ideally slightly shorter at 55-75 characters per line

Text Within Graphics

Limit text in banner graphics to approximately 200 characters for brand identification, a value statement and the call to action. This character count will fit comfortably in the area and allow sufficient reading time in a rotating banner.

Use Kievit, which is our corporate font, or Arial.

In Photoshop, use “crisp” anti-alias setting with a tracking value of 20.

Sample banner graphic with text

Requirements

  • For accessibility, use text rendered as graphics sparingly since it will appear blurry to lowvision users employing magnification tools
  • All graphic text must have the appropriate ALT and title attributes for accessibility
  • Graphics are not to be used for navigation
  • Ensure that text and graphics when viewed without color are accessible, meeting WCAG 2.0 Level AA standards (see Accessibility appendix)

Photography and imagery

Images and photography help Cengage.com maintain a cohesive look.

Images should be unique and impactful:

  • Select simple, bold images with a single focus
  • Choose metaphorical rather than literal images whenever possible
  • Use easy-to-comprehend images that fit the message
  • Use clean backgrounds that don’t distract from the main image

Guidelines for stock photography:

  • Don’t use stock photos of people. Instead, obtain original images of customers, authors, etc. People photos should convey the scope and diversity of the Cengage Learning universe.
  • When using non-people stock photos, avoid clichés. Look for unique or unusual photos.

Home page promotions

The home page may feature one or more large background images. One “hero” with a single focus and message is displayed per visit or screen refresh. The option for a background image is reserved for the home page. HTML text is preferred for accessibility when using a single static image.

Specifications

  • Home page hero images fit 975 px x 470 px, with copy and critical image content in the top 280 px to clear the secondary message overlay band (126 px tall). Images may extend beneath the overlay bar to 2000 px x 1000 px for visual impact on larger screens
  • JavaScript code is used to display one image randomly from a set each time the page reloads or refreshes (see Quick Links appendix)

Secondary pages

A banner graphic conveys a single message; a slideshow script displays multiple messages or images. Page banners should support the page content and don't necessarily have a call to action.

Specifications

  • Secondary pages may have a static image or slideshow that fits the page grid. Full width is 975 px with a suggested depth of 224 px, or with left column navigation, 724 px x 224 px
  • Limit slideshows to a maximum of five slides
  • Headshots, like an author photo, should be 120 px x 150 px

Basic Structure

The page structure includes the header and footer with an underlying grid for the content area.

A corporate home page has a large hero background image and message, with an optional secondary message 126 px tall overlay band at the bottom. This overlay as well as the page footer are anchored to the bottom of the browser window.

Page structure shown with corporate header/footer and home page hero image with overlay band

Note

  • The corporate header and footer are shown. Available header and footer combinations are shown later in this section

Specifications

  • The header and footer background colors extend to the browser window edge, while content is fixed and centered at 975 px wide
  • With the footer anchored to the browser window, deeper page content will scroll while the footer remains stationary and always visible and accessible
  • The optional home page with hero image may also have an overlay band for additional copy with links or thumbnail images of screen captures, videos or book covers
  • If not using the hero image option, the page structure of the home page should follow the same grid as the secondary pages

Content Grid

Grid options for non-corporate pages and any Cengage Learning Marketing websites

The header and footer fill the visible width of the browser window. All other content except for a home page background is limited to the fixed 975 px width. This page grid may also be used for a home page as well as the secondary pages.

Specifications

  • Grid option of 230 px left column for pagelevel navigation, 480 px wide center content column and 230 px wide right column for banners or additional content
  • Grid option of 230 px left column and 724 px wide content (2-column span)
  • Grid option of left 731 px wide content area (2-column span) with 230 px right column for navigation or additional content

Page structure shown with corporate

Left column

Use for secondary or local, page-level navigation.

Main content

The main content may be in the center column, span the left and center columns or the center and right.s

Right column

Use for persistent information like a contact list or a banner ad. A banner ad should expand or support the main page content, not distract from it.

In a layout with content spanning the left and center columns, use the right column for secondary or local, page-level navigation or leave blank.

Specifications

  • Copy can span 2 columns of the grid but should not span all 3 to keep it at a readable maximum line length of 100–120 characters per line, with a preferred count of 55–75 characters per line
  • When content is only in 2 columns with no page-level navigation, use the left and center columns, and leave the right column blank for readable line lengths.
  • Images may run the full 975 px, fit in a single column or span 2 columns
  • If content spans 2 columns, use only left and center or center and right on a single page. Do not mix left spanning and right spanning together on a single page
  • Limit right column banner ads to a maximum of three
  • Right column content should never exceed the depth of the main page content

Footer Options

The corporate footer includes the country selector. Other websites should use the base footer or a full footer. The footer stays anchored to the bottom of the browser window, regardless of the height of the page content. Even if the page content needs to scroll, the footer always remains visible on screen.

Specifications

  • Corporate footer includes the country selector
  • Full, base and branded footers have the option to include social media links
  • The base footer is the minimum recommended links to be included on all websites and pages
  • Branded footer is used if the logo in the header is anything other than Cengage Learning or an imprint logo. The Cengage Learning logo in the branded footer should link to www.cengage.com
  • Use the white level of the footer for secondary, persistent navigation links specific to a website

Local Navigation

The header should contain the primary navigation for a website. A header option that includes a single drop-down menu is available. Space above the tab bank area may display user information, such as signed in account information. The Descriptor text link should link to the home page for the website.

Specifications

  • Use a header with a single drop-down menu for a navigation list to be accessible from anywhere in the website
  • JavaScript for the drop-down menu is available in our code library (see Quick Links appendix)
  • Other header options are a single tab bank or text links directly on the background color
  • Navigation links on a light background are #0071A2 with an underscore for the hover state

Header drop-down menu, available in Tabbed Header with menu option

Breadcrumb

A breadcrumb trail offers users a hierarchical way to navigate or trace back the path they have taken in coming to a web page. Use locationbased breadcrumbs, with text links separated by the “greater than” (>) symbol, and in reverse order of the page title. Company Name > Site Name > Section > Page Name/Description.

Example:

Cengage Learning > K-12 > Trades > Construction

Specifications

  • Breadcrumbs fall below the header and above the page headline
  • Breadcrumbs are a user convenience feature and should be used for multi-level sites to provide additional, alternative navigation
