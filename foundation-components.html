Header and Footer

The following steps will guide you through loading the Cengage Header and Footer onto your HTML page. We have implemented versioning into our headers and footers and the following instructions are based on the versioned header and footer. Replace the <insert-version-here> in the file path with whatever version you wish to load.

Heads up! You are no longer required to have a <header> or <footer> on your page, however, it is still supported. Keep in mind, if you do have those tags on the page they will overwrite the "catalog" setting in the init setup!

Warning! It is recommended to load the latest version only during development. It is not recommended to load the latest version in production since latest points to the most recently added version which may contain changes that break your page.

The following are the current version of the headers:

  • 1.0
  • 1.1
  • latest (1.1)

The Source Files

There are two ways to load the style of the header and footer. The first way is to use javascript to load the styles which is a little slower and may result in a flash of unstyled content. The second is to load the css via style tags on your page.

If you wish to load the styles via the CSS tags on the page (recommended) then copy and paste the below lines of code respectivly

<!-- use this tag if you intend to load the header on your page -->
<link href="//cdn.cengage.com/static/sites/cengage/header-footer/<insert-version-here>/css/cengage-header.css" rel="stylesheet" />

<!-- use this tag if you intend to load the footer on your page -->
<link href="//cdn.cengage.com/static/sites/cengage/header-footer/<insert-version-here>/css/cengage-footer.css" rel="stylesheet" />

<script src="//cdn.cengage.com/js/jquery/jquery-1.7.min.js"></script>
<script src="//cdn.cengage.com/static/sites/cengage/header-footer/<insert-version-here>/js/loader.js"></script>

If you wish to load the styles via the javascript and not a link tag above, then copy and paste these lines in between <head></head> on your HTML page.

<script src="//cdn.cengage.com/js/jquery/jquery-1.7.min.js"></script>
<script src="//cdn.cengage.com/static/sites/cengage/header-footer/<insert-version-here>/js/loader.js"></script>

The Script

The header and footer loaded individually. Before loading the header and footer you must first initialize the loader, this loads the prerequisites shared by both the header and footer. The following are the key components to setting up the script. The values listed below are the defaults for each attribute

loader.init({
  css:{ //use only if you did not load the css yourself
    header:false,        //set true to load header CSS
    footer:false,        //set true to load footer CSS
    fontAwesome: true,   //set to false if you are loading this yourself
    openSans: true       //set to false if you are loading this yourself
  },
  environment: "domain", //load resources via specific environment: "development", "staging" or "production".
  catalog: "pan-cl",     //load content: "pan-cl", "higher-ed", "professional", "gale", or "thorndike". 
  callback:function(){   //callback when the initialization is complete
    //load the header and footer here
  }
});

After initializing the loader, then you are able to load the header or footer or both. Loading the header or footer should be done in the initialize callback function. The following examples assume you are using the default values listed above.

Header and Footer Script

If you want to load both the Header and Footer, copy and paste the script below before </body> tag in the HTML document.

loader.init({
  callback:function(){
    loader.header().footer();
  }
});

Header Only Script

If you want to load only the header copy and paste the script before </body> tag in the HTML document. By default the pan-cl header and pan-cl footer are loaded unless otehrwise specified (explained later in the documentation)

loader.init({
  callback:function(){
    loader.header();
  }
});

Header Utililty Scripts

Header title or subtitle

If you want to a title or subtitle next to the Cengage logo then use this.

loader.init({
  callback:function(){
    loader.header(function(){
      loader.utils.catalogTitle("Title");
      loader.utils.catalogSubtitle("Subtitle");
    }); 
  }
});

Replace Navigation

If you want to replace the navigation section of the header with your own content then you are able to provide a class name and file to load into the nav. The class name will be appended to the nav tag and the content at the URL will be loaded into the nav tag.

Heads up! This method is not supported on the Gale or Thorndike header due to HTML constraints. 

loader.init({
  callback:function(){
    loader.header(function(){
      loader.utils.catalogTitle("Careers");
      loader.utils.nav("class name", "URL to file");
    }); 
  }
});

Remove Navigation

If you want to remove the navigation from the header then use this.

loader.init({
  callback:function(){
    loader.header(function(){
      loader.utils.removeNav();
    }); 
  }
});

Footer Only Script

If you want to load only the footer copy and paste the script before </body> tag in the HTML document.

loader.init({
  callback:function(){
    loader.footer(); 
  }
});

Footer Utility Script

Pan-CL footer shown in examples below, however, all utility scripts work on all footers.

Custom Footer Logo

Use this script if you want to load custom image into the footer.

Replace the //cengage.com/images/example-image.png with your unique image link.

loader.init({
  callback:function(){
    loader.footer(function(){
      loader.utils.logo("//www.cengage.com/images/example-image.png");
    }); 
  }
});

Hide Country in Footer

Use this script if you want to hide the country in the Cengage footer.

loader.init({
  callback:function(){
    loader.footer(function(){
      loader.utils.hideCountry(); 
    }); 
  }
});

Show the Sign-In Band

Use this script if you want to show the sign-in band. The footer Sign-In Band requires the footer tag to be present but does not require the footer to be loaded before it can be loaded. This is why the call to the utils is not placed in the footer callback. there to be a footer tag present but does not need

loader.init({
  callback:function(){
    loader.footer(); 
    loader.utils.signIn(); 
  }
});

Video Player

Vidyard

This plugin makes the process of using vidyard easier. The plugin will accept a JSON or XML file. The plugin also adds the ability to add thumbnails for multiple videos to be used. We have implemented versioning for the vidyard plugin. Replace the <insert-version-here> in the file path with whatever version you wish to load.

Warning! It is recommended to load the latest version only during development. It is not recommended to load the latest version in production since latest points to the most recently added version which may contain changes that break your page.

The following are the current version of the vidyard plugin:

  • 1.0
  • 1.1
  • latest (1.1)

Change Log

1.1 changes

  • Videos are now dynamically resized removing black borders and maintaining 16:9 aspect ratio
  • Loading overlay removed on mobile devices to fix issue with the overlay covering videos after initialization.

Getting Started

The prerequisites for the plugin are:

jQuery (at least version 2.0)

<script src="//cdn.cengage.com/js/jquery/jquery-2.1.4.min.js"></script>

If multiple tabs then jQuery UI is required:

<script src="//cdn.cengage.com/js/jquery-ui/jquery-ui-1.11.4.min.js"></script>

The Plugin:

<script src="//cdn.cengage.com/js/vidyard/<insert-version-here>/jquery.vidyard-plugin.js"></script>

The CSS to style the Video:

<link rel="stylesheet" type="text/css" href="//cdn.cengage.com/js/vidyard/<insert-version-here>/video-style.css"/>

Example

Once you have the prerequisites in place it should look like this:

<link rel="stylesheet" type="text/css" href="//cdn.cengage.com/js/vidyard/<insert-version-here>/video-style.css"/>
<script src="//cdn.cengage.com/js/jquery/jquery-2.1.4.min.js"></script>
<script src="//cdn.cengage.com/js/jquery-ui/jquery-ui-1.11.4.min.js"></script>
<script src="//cdn.cengage.com/js/vidyard/<insert-version-here>/jquery.vidyard-plugin.js"></script>

Implementation

There must be a container in the body of the document with a couple of attributes to instantiate the player:

<div class="video-container vidyard-video" id="video" player="vidyard1"></div>

The “class” attribute is required if you wish to keep the pre-styled player and tabs. However, it can be removed if you wish to style the player yourself.

The “id” attribute is not required but I am using it to select the element to initilize the plugin on.

To initiate the plugin a couple lines of JavaScript are required. The player accepts JSON for single videos and XML for multiple or single videos.

To Remove H Tags

Use the description field insted of title if you do not want the h tags to appear

Example

Here is an example of all the parameters the single player plugin accepts:

$("#video").player({
    json: {
        uuid: "uuid of video on vidyard",
        description: "description of video",
        headline: "Title of video",
    },
    details: true or false (defaults to true), // displays captions
    callback: "function to be called when the player is fully setup" 
});

Multi video player

If you wish to use a multi video player then an XML file is required. Here is an example of a multi video player:

$("#video").player({
    xml: "http://mydomain.com/path/to/xml",
    details: true, // displays captions (defaults to true)
    single: false, //true will display first video if multiple are present (defualts to false)
    columns: 4, //number of thumbnail columns (defaults to 4)
    callback: function(){} //function to be called when the player is fully setup
});

JavaScript does not allow cross domain loading, if you are trying to load an xml from a different domain that does not allow CORS the video player will not work.

If an XML only has one video then it will default to a single video player

Jwplayer (no longer supported)

This plugin makes the process of using jwplayer easier. The plugin will accept a JSON or XML file. The plugin will always instantiate the player with flash as the primary player. If flash has not been installed in the user’s browser then HTML 5 video player will be used. If the browser does not support flash or HTML 5, a download fallback will be used.

Getting Started

The prerequisites for the plugin are:

jQuery

<script src="//cdn.cengage.com/js/jquery/jquery-1.7.min.js"></script>

jwplayer

<script src="//cdn.cengage.com/js/jwplayer/6.11/jwplayer.js"></script>

If using JQuery tabs is required:

<script src="//cdn.cengage.com/js/contrib/jquery.a11y.ui/ui/jquery.ui.widget.js"></script>
<script src="//cdn.cengage.com/js/contrib/jquery.a11y.ui/ui/jquery.ui.tabs.js"></script>

The Plugin:

<script src="//cdn.cengage.com/js/contrib/jquery.jwplayer-impl.js"></script>

The CSS to style the Video:

<link rel="stylesheet" type="text/css" href="//cdn.cengage.com/css/cl/video-style.css"/>

Example

Once you have the prerequisites in place it should look like this:

<script src="//cdn.cengage.com/js/contrib/jquery.a11y.ui/ui/jquery.ui.tabs.js "></script>
<script src="//cdn.cengage.com/js/contrib/jquery.jwplayer-impl.js "></script>

Implementation

There must be a container in the body of the document with a couple of attributes to instantiate the player:

<div class="video-container" id="video" player="jwplayer1"></div>

The “class” attribute is required if you wish to keep the pre-styled player and tabs. However, it can be removed if you wish to style the player yourself.

The “id” attribute can be whatever you want.

The “player” attribute is required to give the jwplayer an id. You can give this attribute any value but it must start with “jwplayer” (jwplayer, jwplayer1, jwplayer2, jwplayerContainer, etc.)

To initiate the plugin a couple lines of JavaScript are required. The player accepts JSON for single videos and XML for multiple videos.

To Remove H Tags

Use the description field insted of title if you do not want the h tags to appear

Example

Here is an example of all the parameters the single player plugin accepts:

$("#video").player({
    json: {
        mp4: "http://myDomain.com/path/to/mp4",
        webm: "http://myDomain.com/path/to/webm",
        srt: "http://myDomain.com/path/to/srt",
        desc: "description of video",
        title: "Title of video",
        youtube: "youtube video",
        img: "path to image file",
        txt: "transcript file to download",
        callback: "function to be called when the player is fully setup" 
    },
    key: "jwplayer license key",
    stream: true or false (defaults to false),
    height: "height of the player",
    width: "width of the player",
    details: true or false (defaults to true), // displays captions
    loop: true or false (defaults to false) // makes video will loop
});

Multi video player

If you wish to use a multi video player then an XML file is required. Here is an example of a multi video player:

$("#video").player({
    xml: "http://mydomain.com/path/to/xml",
    key: "jwplayer license key",
    stream: "true or false (defaults to false)",
    height: "height of the player",
    width: "width of the player",
    callback: "function to be called when the player is fully setup",
    details: true or false (defaults to true), // displays captions
    single: true or false (defaults to false), //true will display first video if multiple are present.
    loop: true or false (defaults to false), // makes video will loop
    columns: number of columns (defaults to 4)
});

The XML file can have these attributes and any of them can be omitted if you choose. However, there must be at least one video URL (mp4, webm or youtube). Also, JavaScript does not allow cross domain loading, if you are trying to load an xml from a different domain that does not allow CORS the video player will not work.

If an XML only has one video then it will default to a single video player

Forms

Requirements

  • Most forms should be Eloqua-based
  • If the primary purpose of the web page is the form, auto-focus on the first form field on the page when the page loads
  • Forms should always be one column with labels placed to the left of form fields
  • All form fields must have labels that clearly convey what information the field is collecting
  • Forms must have tabbing order set for screen readers and users who do not use a mouse to sequentially navigate through a form
  • Auto-suggest text fields for long pick lists rather than offering drop-downs is preferred
  • Use placeholder text to provide additional content hints
  • Placeholder text must have its own label for accessibility
  • Error messages appear inline in #CE3C23
  • Use an action verb describing the purpose of the form for the button label rather than “Submit.”
    For example, on an event registration form, the button should say “Register”; on an e-mail subscription form, “Subscribe.”

Form Sample

Form Error Messages

Forms with required fields must have the following notation at the top of the form:

Required fields are indicated with *.

Labels for required fields must include a red asterisk, that does not interfere with screen readers.
A font glyph, such as Font Awesome, is suggested (see Quick Links appendix). 

<div>
  <label for="firstName">
    <i class="iconasterisk"></i>
  </label>
  <input type="text" name="firstName" id="firstName" aria-required="true">
</div>

When tabbing through the form, a screen reader will announce “First name, edit field, required.”

A fieldset is required for radio buttons. The fieldset legend clearly describes what the user will be selecting.
For example:

<fieldset>
  <legend>Email Frequency</legend>
  <label for=“d”>Daily</label> 
  <input type=“radio” name=“freq” id=“d”>
  <label for=“w”>Weekly</label> 
  <input type=“radio” name=“freq” id=“w”>
  <label for=“m”>Monthly</label> 
  <input type=“radio” name=“freq” id=“m”>
</fieldset>

Screen readers will announce this as “e-mail frequency, daily, radio button unchecked;
e-mail frequency, weekly, radio button unchecked;
e-mail frequency, monthly, radio button unchecked.”
